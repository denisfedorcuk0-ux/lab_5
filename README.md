## Опис реалізації
# Toy (Абстрактний базовий клас)
Абстрактний базовий клас для всіх іграшок у кімнаті.

Поля: name (назва), price (ціна), ageGroup (вікова група: маленькі, середні, великі).

Методи: геттери для всіх полів, перевизначений toString().

# Car / Doll / Ball / Blocks (Конкретні класи-нащадки)
Класи, що наслідуються від Toy та представляють конкретні типи іграшок.

Car: Додатково має поле size (розмір машинки: маленька, середня, велика).

Doll: Додатково має поле material (матеріал, з якого виготовлена лялька).

Ball: Додатково має поле diameter (діаметр м'яча в сантиметрах).

Blocks: Додатково має поле piecesCount (кількість елементів у наборі кубиків).

# PlayroomException (Клас винятку)
Кастомний клас винятку для обробки помилок бізнес-логіки.

Використовується при перевищенні ліміту виділеного бюджету, передачі некоректних (від'ємних) цін або неправильних меж пошуку.

# Room (Клас для роботи з ігровою кімнатою)
Клас, що керує колекцією іграшок у кімнаті.

Створення масиву: Toy[] з фіксованою кількістю іграшок різних вікових груп.

Обчислення вартості кімнати: calculateTotalCost() для перевірки відповідності виділеній сумі грошей.

Сортування іграшок: sortByPrice() для сортування за параметром вартості.

Пошук іграшки: findToysByPriceRange() для знаходження іграшок, що відповідають заданому діапазону вартості.

# Main (Клас для демонстрації роботи)
Головний клас, що запускає програму.

Ініціалізація масиву іграшок різних типів та вікових груп.

Створення об'єкта кімнати з перевіркою бюджетного ліміту та обробкою виняткових ситуацій (try-catch).

Виклик методів сортування за ціною та пошуку іграшок у заданому діапазоні вартості з виведенням результатів у консоль.
